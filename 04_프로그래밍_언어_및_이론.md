
# 🧾 04_프로그래밍_언어_및_이론

프로그래밍 언어 및 컴퓨터 이론 파트는 자바(Java)를 중심으로 변수의 타입, 컴파일 방식, 객체지향 개념 등을 설명할 수 있는 능력을 평가합니다. 명확한 개념 정의와 함께 코드 레벨 이해가 중요합니다.

---

## 1. 변수의 타입 지정이 가지는 의미는 무엇이며, 정적 타입과 동적 타입 언어의 차이는 무엇인가요?

- **타입 지정**은 변수에 저장될 데이터의 종류를 명확히 하고, 컴파일러나 인터프리터가 오류를 사전에 잡을 수 있게 해줍니다.

| 구분       | 정적 타입 언어 (Static Typing) | 동적 타입 언어 (Dynamic Typing) |
|------------|------------------------------|-------------------------------|
| 시점       | 컴파일 타임                   | 런타임                        |
| 예시       | C, Java                      | Python, JavaScript           |
| 장점       | 성능 우수, 오류 조기 발견 가능     | 유연성 높음, 개발 속도 빠름          |
| 단점       | 타입 선언 필요, 유연성 부족        | 실행 중 오류 발생 가능             |

---

## 2. 컴파일러와 인터프리터의 차이를 설명해 주세요.

| 항목         | 컴파일러 (Compiler)         | 인터프리터 (Interpreter)         |
|--------------|-----------------------------|----------------------------------|
| 실행 방식     | 전체 소스를 기계어로 변환 후 실행   | 한 줄씩 읽고 즉시 실행               |
| 성능         | 빠름                         | 느림                              |
| 오류 검출 시점 | 컴파일 타임                   | 실행 시점                           |
| 예시         | C, C++, Java (바이트코드)    | Python, JavaScript, Ruby         |

---

## 3. 시간복잡도와 공간복잡도의 정의와 이를 분석하는 방법에 대해 설명해 주세요.

- **시간복잡도(Time Complexity)**: 입력 크기(n)에 따라 알고리즘이 소요하는 연산 횟수의 증가율
- **공간복잡도(Space Complexity)**: 입력 크기에 따라 추가적으로 필요한 메모리 양

### 주요 표기법:
- **Big-O (최악의 경우)**: O(n), O(log n), O(n²) 등
- **분석 방법**: 루프, 재귀 호출, 분할 정복 구조 등 파악하여 추상적 연산 횟수 도출

## 4. 자주 사용되는 디자인 패턴 중 하나를 선택해 개념과 사용 사례를 설명해 주세요.

### 예: 싱글턴 패턴 (Singleton Pattern)
- **의미**: 클래스의 인스턴스를 하나만 생성하여 전역에서 공유하도록 하는 패턴
- **활용 예시**: DB 연결, 설정 정보 관리, 로그 기록 객체 등
- **장점**: 자원 절약, 일관된 상태 유지  
- **주의점**: 멀티스레드 환경에서 동기화 문제

## 5. 자바(Java)에서 다중 상속이 불가능한 이유는 무엇이며, 이를 해결하기 위한 방법은 무엇인가요?

- Java에서는 **클래스 간 다중 상속을 금지**하여 **Diamond Problem**을 방지합니다.
- 대신 **인터페이스를 다중 구현**함으로써 다중 상속 유사 구조를 구현합니다.

```java
interface A { 
  void print(); 
}

interface B { 
  void show(); 
}


class C implements A, B {
    public void print() { ... }
    public void show() { ... }
}
```

## 6. 객체(Object)와 클래스(Class)의 차이점을 설명해 주세요.

- **클래스**: 객체를 생성하기 위한 설계도 또는 틀  
- **객체**: 클래스에 의해 생성된 실체로, 메모리에 할당된 인스턴스

예시:  
`class Car { ... }` → `Car myCar = new Car();` (객체 생성)

---

## 7. 자바에서 클래스와 인터페이스의 차이점은 무엇이며, 각각을 언제 사용하는 것이 적절한가요?

| 항목         | 클래스(Class)                 | 인터페이스(Interface)           |
|--------------|-------------------------------|----------------------------------|
| 목적         | 구현(기능과 상태 정의)            | 규약(기능의 명세 정의)              |
| 멤버 구성     | 변수, 메서드, 생성자               | 상수, 추상 메서드, default 메서드 등 |
| 상속/구현     | 단일 상속                       | 다중 구현 가능                       |
| 인스턴스 생성 | 가능                           | 직접 생성 불가                        |

- **클래스**: 공통 기능을 구현할 때  
- **인터페이스**: 확장성과 유연성을 위한 표준 명세 정의 시

## 8. 오버라이딩(Overriding)과 오버로딩(Overloading)의 차이점과 사용 예시를 설명해 주세요.

| 항목         | 오버라이딩 (Overriding)             | 오버로딩 (Overloading)              |
|--------------|-------------------------------------|-------------------------------------|
| 정의         | 상속받은 메서드를 재정의                | 같은 이름의 메서드를 매개변수만 다르게 정의 |
| 매개변수      | 동일해야 함                            | 다르게 정의 가능                        |
| 리턴 타입     | 동일하거나 하위 타입이어야 함              | 상관 없음                              |
| 위치         | 상속 관계 클래스                        | 같은 클래스 내부                         |

## 9. 객체지향 언어의 특징은 무엇이며, 절차지향 언어와의 차이는 무엇인가요?

### 객체지향 특징 (OOP 4대 특성):
1. **추상화(Abstraction)**  
2. **캡슐화(Encapsulation)**  
3. **상속(Inheritance)**  
4. **다형성(Polymorphism)**

| 항목       | 절차지향 언어                  | 객체지향 언어                       |
|------------|-------------------------------|-------------------------------------|
| 단위       | 함수 중심                      | 객체 중심                            |
| 재사용성   | 낮음                          | 높음                                 |
| 유지보수   | 어렵다                         | 비교적 용이함                         |

## 10. 추상 클래스와 인터페이스의 개념과 차이를 설명해 주세요.

| 항목         | 추상 클래스(Abstract Class)         | 인터페이스(Interface)              |
|--------------|--------------------------------------|-------------------------------------|
| 목적         | 공통 기능 제공 + 일부 구현 포함 가능     | 구현 없이 명세만 제공                   |
| 다중 구현     | 불가 (단일 상속만 가능)                | 가능 (여러 인터페이스 구현 가능)         |
| 사용 시점     | 기능 일부 구현 제공하고 싶을 때            | 표준 명세 정의하고 싶을 때                |

---

## 11. 다형성(Polymorphism)이란 무엇이며, 자바에서 어떻게 구현되는지 설명해 주세요?

- **다형성**: 동일한 인터페이스를 통해 서로 다른 동작을 수행할 수 있는 능력
- **Java에서의 구현 방식**:
  - **오버라이딩 (상속 기반 다형성)**
  - **인터페이스 구현**
  - **업캐스팅(부모 타입 참조로 자식 객체 제어)**

예시:
```java
Animal a = new Dog();
a.sound(); // Dog의 sound() 실행
```
